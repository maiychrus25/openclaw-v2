**Cancel Enrollment — Root Cause Analysis:**

Sau nhiều lần fix, tìm ra root cause thực sự:
1. **BE port 5002 đang chạy code cũ** (ts-node-dev process PID 20816) — fix không có tác dụng vì process cũ vẫn active trên port
2. **`$lookup` filter `status: { $ne: 'canceled' }` quá aggressive** — lọc luôn enrollment đã hủy khỏi `latest_enrollment` → data KSK + progress biến mất hoàn toàn khỏi UI
3. **Status badge dùng `status_detail` thay vì `enrollment.status`** — không nhận diện được 'canceled' → hiển thị "Đang học"

**Fix lần 3 (triệt để):**
- BE: Bỏ filter `$ne: 'canceled'` ở main `$lookup` → canceled enrollment vẫn hiện trong `latest_enrollment`
- FE: Badge ưu tiên `enrollment.status` trước `status_detail` → "ĐÃ HỦY" hiển thị đúng
- FE: `isEnrollmentCanceled = enrollment.status === 'canceled'` → guards hoạt động đúng
- Tất cả data (KSK, progress, thanh toán) vẫn hiển thị read-only
- Thầy giáo không sửa được khi đã hủy

**MRs đã update:**
- !69 BE: `831d2a8` (lookup filter + course null) → `461cc01` (include canceled)
- !74 FE: `c314a27` (guards + banner) → `dc551a0` (status badge fix)

**Lessons Learned:**
- 2 BE processes cùng chạy → phải kill đúng PID đang listen trên port
- `$lookup` filter quá aggressive có thể làm mất data quan trọng
- Status badge: enrollment-level status (`canceled`) override course-level status (`status_detail`)
